## Wikivoyage raw data collection

Summary:
- Created fetch_wikivoyage_raw.py script to collect real destination content from Wikivoyage.
- Script fetches from Wikivoyage MediaWiki API (primary) with direct HTML page fallback (secondary).
- Proper User-Agent header per Wikimedia terms of service.
- HTML cleaning: removes [citations], [edit] links, normalizes whitespace.
- Splits each destination into 2 documents (_overview.txt + _details.txt) for balanced retrieval.
- Fixed API response parsing: correctly extracts text from `data["parse"]["text"]["*"]` structure.
- Successfully collected and ingested 20 real Wikivoyage documents (10 destinations × 2 sections).
- Generated ~1,800 chunks with embeddings stored in Postgres+pgvector.
- All source files include proper attribution: destination, source_title, and source_url headers.
- Updated docs/rag_foundation.md with "Raw Corpus Collection" section explaining the real-sources-only approach.
