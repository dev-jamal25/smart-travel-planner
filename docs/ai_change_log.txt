## LangSmith custom tracing integration

Summary:
- Added langsmith>=0.7.37 as an explicit dependency (was already a transitive dep of langchain-anthropic).
- Created app/tracing.py with three pre-configured @traceable decorators:
  plan_trip_trace (top-level chain), model_call_trace (LLM method chain), tool_call_trace (tool).
- All decorators use process_inputs to strip non-serialisable / sensitive parameters before
  any data is sent to LangSmith: self (contains API key), session, run_id, service objects.
  System prompts, auth tokens, and webhook URLs are never part of traced function arguments.
- Decorated eight functions:
  AgentService.plan_trip (top-level), ModelRouter.extract_trip_intent, rewrite_rag_query,
  select_candidate_destination, synthesize_final_answer; destination_knowledge_retrieval,
  classify_destination_style, fetch_live_weather.

  if langchain_tracing_v2=True, sets LANGCHAIN_TRACING_V2/LANGCHAIN_API_KEY/LANGCHAIN_PROJECT
  via os.environ.setdefault so Docker/shell values take precedence.
- Updated tests/conftest.py: force LANGCHAIN_TRACING_V2=false in test suite to prevent
  any accidental LangSmith network calls.
- When tracing is disabled, decorated functions behave exactly as if undecorated.
- LangGraph also emits native LangSmith node traces automatically when env vars are set.
- README screenshot of a LangSmith trace must be captured manually after one real query.

## Fix Supabase JWT verification to support ES256 tokens via JWKS

Summary:
- Supabase issues ES256-signed JWT tokens, not HS256.
- Previous implementation tried to verify ES256 tokens with HS256 algorithm + symmetric secret, causing "Invalid or expired token" errors on all protected endpoints.
- Updated get_current_user() in app/dependencies.py to use Supabase JWKS (JSON Web Key Set) for public-key verification.
- Uses PyJWT's PyJWKClient to fetch public keys from Supabase JWKS URL and verify ES256 tokens.
- Caches JWKS client with lru_cache(maxsize=1) to avoid refetching on every request.
- Validates audience, issuer, and expiration as before.
- Maintains HTTPBearer security scheme for Swagger auth display (lock icon still shows).

Files affected:
- backend/app/config.py (replaced supabase_jwt_secret with supabase_jwt_jwks_url)
- backend/app/dependencies.py (added _get_jwks_client(), updated get_current_user() to use PyJWKClient)
- backend/tests/test_auth.py (updated tests to mock JWKS client instead of using HS256)
- .env.example (replaced SUPABASE_JWT_SECRET with SUPABASE_JWT_JWKS_URL)
- docs/ai_change_log.txt (this entry)

Commands executed:
- uv run ruff check . (passes)
- uv run mypy app/ (passes)
- uv run pytest -q (all tests pass with JWKS mocking)

Verification:
- Missing auth: returns 401 AuthError

## Weather/live-conditions tool foundation

Summary:
- Added weather forecast service backed by Open-Meteo (no API key required).
- WeatherService is a lifespan singleton: one httpx.AsyncClient, one TTLCache per process.
- Cache TTL is configurable via WEATHER_CACHE_TTL_SECONDS (default 600 s); TTLCache(maxsize=128).
- Tenacity AsyncRetrying: 3 attempts, wait_exponential(0.5 s → 8 s), retries only on
  httpx.RequestError (network failures); HTTP status errors propagate immediately.
